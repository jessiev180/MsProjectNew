# Bake Project - 烘焙平台資料庫

這是本專案的資料庫核心腳本存放處。包含了會員、電商、社群、客服及平台管理五大模組，總計 44 張資料表。

## 📁 資料夾結構說明
* `/Database`: 存放所有 SQL 建置腳本。
* `/Models`: (後續更新) 存放對應資料庫的 C# POCO 類別。

## 🚀 資料庫建置步驟 (重要！)
為了確保資料庫關聯正確，請務必按照以下順序在 SSMS 執行腳本：

1. **執行 `Master_Setup.sql`**
   - 建立 `Bake` 資料庫。
   - 建立邏輯分層 Schema (`[User]`, `Sales`, `Social`, `Servers`, `Platform`)。

2. **執行 `All_Tables_v1.sql`**
   - 建立全站 44 張資料表。
   - 建立所有 Foreign Key 關聯與限制條件 (Constraints)。

## 🛠 開發規範
- **Schema 命名**：會員相關請使用 `[User]`，電商請使用 `Sales`。
- **命名慣例**：
    * 表名 (Tables): 資料表採用底線分隔 開頭大寫Snake_Case。使用「名詞」，建議採單數形式（例如 User 而非 Users）。
    * 欄位名 (Columns): 採用底線分隔 snake_case，並保持一致。 (目前我們組大家是使用snake_case，請檢察自己欄位是否一致)
    * 不要使用 sp_ 前綴: 這是微軟官方多次強調的。sp_ 是保留給「系統預儲程序」的，若自訂程序以此開頭，SQL Server 會先去 master 資料庫搜尋，導致效能下降。
    * 不要使用 tbl_ 前綴: 微軟建議不要在表名前加 tbl_ 或 t_（匈牙利命名法），因為物件類型已經在 Metadata 中定義了，重複標註會顯得冗餘。

- **變動通知**：若有修改資料表結構，請更新 SQL 腳本後推送到 GitHub 並通知組員更新。
